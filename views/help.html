1. 개요[편집]

이 문서는 나무위키의 위키 문법인 '나무마크'에 대하여 설명하는 문서입니다. 나무위키의 나무마크는 모니위키의 마크업과 100% 호환되지는 않습니다. 일부 기능은 모니위키에서 작동하던 것과 다르게 작동하는 경우가 있습니다. 모니위키에서 사용하던 일부 마크업은 계속 지원하고 있지만, 앞으로는 나무위키의 문법에 맞도록 수정 및 작성해주시기 바랍니다. 모니위키 호환 마크업은 예고 없이 지원이 중단될지도 모릅니다.

2. 모니위키 문법과 호환성[편집]

  1. <<매크로>> 형태를 지원하지 않습니다. [매크로] 형태로 작성하여 주십시오.

  2. 문서 내에 HTML 코드를 직접 사용해야 하는 경우, {{{#!html HTML코드}}}의 형태로 사용해주시기 바랍니다. (비권장 문법)

  3. $ LaTeX $ 형태의 수식 입력을 지원하지 않습니다. [math(LaTeX)] 형태로 작성하여 주십시오.

  4. [문서이름] 이나 [[wiki:"문서이름"]] 등 기타 유사 형태의 링크를 지원하지 않습니다. [[문서이름]] 으로 작성하여 주십시오.

  5. - 문자는 네 개 이상, 아홉 개 이하로 써야 수평줄로 랜더링됩니다.

  6. 문단 제목의 경우 = 7개 이상의 깊이를 쓸 수 없습니다.

  7. 다수의 매크로가 사라지거나, 기능이 축소되었습니다.

  8. 모니위키의 경우 리스트문법을 섞어쓸 수 없습니다. 또한 리스트간에 두줄 이상의 공백을 넣어도 여전히 이전의 리스트에 이어지는 순서로 번호가 붙어 랜더링됩니다. (모인모인 문법 호환)

  9. 모니위키의 경우 [[파일:이름]] 형태는 지원하지 않습니다. 모니위키식으로 한다면 [[파일(이름)]] 형식으로 써야 합니다.

  10. 표 문법에서 ||<#000000> 식으로 배경을 넣는 문법은 지원이 중단될 예정입니다. ||<bgcolor=#000000> 으로 써 주십시오.

3. 문단[편집]

문단 제목을 생성하는 방법은 다음과 같습니다. 보통 글자 크기가 제목과 동일한 = 문단 = 보다는 == 문단 ==을 더 많이 사용합니다.

= 문단 1 =

== 문단 2 ==

=== 문단 3 ===

==== 문단 4 ====

===== 문단 5 =====

====== 문단 6 ======

문단의 내용을 기본적으로 숨기는 방법은 다음과 같습니다.

=# 문단 1 #=

==# 문단 2 #==

===# 문단 3 #===

====# 문단 4 #====

=====# 문단 5 #=====

======# 문단 6 #======

주의: =과 문단 제목, 문단 제목과 =의 사이에는 띄어쓰기가 반드시 있어야 합니다. 띄어쓰기를 하지 않을 경우 출력되지 않습니다.

주의: 문단의 단계를 정확히 입력하여 주십시오. 첫 문단을 == 문단 2 ==로 시작했다면 문서 전체에서 = 문단 1 =은 사용되어서는 안됩니다. == 문단 2 ==의 하위 문단은 ==== 문단 4 ====가 아닌 === 문단 3 ===입니다. == 문단 2 == → ==== 문단 4 ==== → == 문단 2 == 같은 형식으로 문서를 작성하면 정상적인 출력을 보장하지 않으며, 이렇게 비정상적인 문단을 사용하면 오류가 발생하기 쉽습니다.

주의: == 문단 2 ==에서== 문단 2 == 형식으로 띄어쓰기가 있는 등, =바로 뒤에 스페이스바로 띄어쓰기가 되어 있으면 출력되지 않는 오류가 발생할 수 있습니다.

4.3. 텍스트 색상[편집]

텍스트의 색상을 조절할 수 있습니다.

{{{#RRGGBB 텍스트}}}: FF0000 or F00

{{{#name 텍스트}}}: RED

#이 색깔 이름 앞에 반드시 들어가야 한다.
RGB 값, 또는 CSS에서 사용할 수 있는 웹 색상을 이용할 수 있습니다. 대소문자는 구별하지 않습니다.
색상값은 웹 색상, 색상 목록을 참고하시기 바랍니다.

RGB에서 앞에 두자리는 R 그 뒤 두 자리는 G 그 뒤 두 자리는 B의 강도를 수치화 한 것이다.
R=red
G=green
B=blue
수치는 10진법이 아닌 16진법을 사용 하며 00이 동작없음 FF가 가장 강하게 이다.

참고: 밑줄 서식과 혼용할 때 두 가지 방법을 쓸 수 있습니다.

  • {{{#green __밑줄 포함 녹색__}}}: 밑줄 포함 녹색

  • __{{{#green 밑줄 제외 녹색}}}__: 밑줄 제외 녹색

  • 위에서 아래 to bottom

  • 아래에서 위: to top

  • 남서쪽 방향 대각선: to bottom left 또는 to left bottom

  • 북서쪽 방향 대각선: to top left 또는 to left top

5. 하이퍼링크[편집]

5.1. 링크와 출력이 같은 하이퍼링크[편집]

링크될 문서와 페이지에 출력되는 내용이 같도록 링크하는 방법은 다음과 같습니다.

[[문서]]


문서명에 [ 또는 ]가 들어가는 경우, \[와 \]로 대체해야 합니다. \를 입력해야 하는 경우, \\로 대체해야 합니다. 마찬가지로 \[를 입력해야 하는 경우 \\[로, \\를 입력해야 하는 경우 \\\\로 대체하여 주십시오.

주의: 모니위키에서 사용됐던 [문서] 형태, 즉 괄호 한 개를 사용하는 링크는 지원하지 않습니다.

5.2. 링크와 출력이 다른 하이퍼링크[편집]

링크될 문서와 페이지에 출력되는 내용이 다르도록 링크하는 방법은 다음과 같습니다.

[[문서|출력]]


|는 shift키와 \키(백슬래시)를 동시에 눌러서 입력할 수 있습니다.
키보드에 /키는 있는데 \키가 보이지 않으면, shift키와 ￦키를 같이 누르거나 alt키를 누른 채로 넘패드(숫자패드)의 124를 입력해도 됩니다. 아니면 위 예시에 있는 | 를 복사해 가서 붙여도 됩니다. 모바일은 기종에 따라 기호에 있습니다.

이 형태에서 문서명에 |가 들어가는 경우, \|로 대체해야 합니다. 출력되는 내용에는 \를 붙이지 않으셔도 됩니다.

주의: 문서와 |, |와 출력 사이에는 공백을 삽입하지 않습니다.
주의: 기존에 사용되던 [[문서|]], 또는 [[|문서]] 형태의 링크는 경우에 따라 지원되지 않을 수도 있습니다. 이는 추후에 지원이 중단될 수 있으므로 권장하지 않습니다. [[문서]] 또는 [[문서|출력]]의 형태로 링크하여 주십시오.
주의: 모니위키에서 사용됐던 [wiki:"문서"] 등의 링크 문법은 지원하지 않습니다.

[[../]]: 상위 문서

[[../|상위]]

[[/문서]]: 하위 문서

주의: 상위, 하위 문서는 실제 상하위 문서가 아닌, 문서명에 쓰인 /를 기준으로 합니다. 예를 들어, S/W 개발병 문서의 상위 문서는 군사특기이지만 이 문서에서 [[../]]를 사용하면 슬래시와 뒷부분을 지운 S 문서로 링크가 갑니다. 또 /가 문서명에 없는 문서에서 [[../]]를 사용하면 그냥 링크를 건 문서 자기 자신으로 링크가 걸립니다.

5.7. |, \, #가 제목에 쓰이거나 제목이 ]로 끝나거나 제목이 /로 시작하는 문서로 하이퍼링크 걸기[편집]

기술적인 문제로 일반적인 하이퍼링크 문법으로는 해당 문서로 통하는 정상적인 하이퍼링크를 만들 수 없습니다. 이럴 때는 해당 문자 앞에 \를 넣어 줘야 합니다. #나 맨 끝의 ]가 제목에 들어간 문서에 한해서 맨 끝에 #를 넣어 줘도 됩니다. 다른 편집자를 위해서 틀:링크시 주의를 달아 줘도 됩니다. 해당 틀을 작성할 때 위키 문법상 \를 3개 더 입력해야 안내문에 \가 정상적으로 출력이 되니 유의하십시오.
단, /는 예외적으로 \를 앞에 입력해도 정상 출력이 안 됩니다. 이때는 링크를 걸려는 쪽의[7] 문서의 제목에 /가 없다면 [[../문서명]]을 사용하십시오.
링크를 걸려는 쪽의 문서 제목에 /가 있을 경우에는 (슬래시(/) 개수+1)개만큼 ../를 써주시고 문서명을 붙이시면 됩니다. 그러나 이 경우 정상적으로 링크되지만 빨간 링크가 걸리므로, 색상 문법을 이용해서 #0275D8의 색을 넣어주시면 정상 색으로 나옵니다.
혹은 모양이 비슷한 전각 문자(＃, ／, ＼, ｜, ］)로 리다이렉트하셔도 됩니다. 전각 문자가 아니더라도 해당 문서의 리다이렉트 문서를 사용하실 수도 있습니다.

6. 리다이렉트(넘겨주기)[편집]

리다이렉트 문서는 한 문서를 검색할 때 목적지 문서로 즉시 이동시켜주는 문서입니다. 넘겨주기 문서를 만드는 방법은 다음과 같습니다.

  1. 먼저 넘겨줄 목적지 문서를 고릅니다.

  2. 넘겨질 출발지 문서로 이동합니다. 주소창에 namu.wiki/delete/생성할 리다이렉트 문서 로 접속하거나 검색창에 검색한 후 '문서로 가기'버튼을 클릭합니다.

  3. 적절한 리다이렉트 문법을 삽입합니다.

    • 문서로 넘기는 경우: #redirect 목적지 문서 또는 #넘겨주기 목적지 문서[10]의 형태로 새 문서를 작성합니다.

    • 문단으로 넘기는 경우: #redirect 목적지 문서#s-(문단 번호) 의 형태로 새 문서를 작성합니다.

    • 특정 앵커로 넘기는 경우: #redirect 목적지 문서#앵커명의 형태로 새 문서를 작성합니다. 목적지 문서에 해당 앵커가 미리 존재하니 주의해 주세요.


문법 변경으로 인하여 생겨난 또 다른 방법이다.

  1. 먼저 넘겨줄 목적지 문서를 고릅니다.
    2.목적지 문서의 편집창에 들어간다.
    3. [[넘겨주기:생성할 리다이렉트]]문법을 문서상단에 적고 저장한다.

    • 특정 문단으로 넘기는 경우: 해당 문단에 같은 문법을 적는다.



주의: 모니위키에서 비권장 문법이던 URL 리다이렉트는 나무위키에서 지원하지 않습니다.
주의: 앵커 리다이렉트의 구현으로 인해 목적지 문서의 표제어에 #(맨 뒤 제외)가 포함되면 해당 문자들의 앞에 \를 삽입해야 합니다.

리다이렉트는 주소창에 namu.wiki/delete/삭제할 리다이렉트 문서로 접속 후 삭제하거나, 검색창에 리다이렉트 문서를 쳤을 때 목적지 문서 상단에 뜨는 리다이렉트 문서를 클릭 후 삭제하실 수 있습니다.

7.1. 나무위키 자체 이미지 업로더[편집]

나무위키의 자체 이미지 업로더를 이용하려면 반드시 문서 관리 방침을 따라야 합니다. 나무위키 사이트 상단의 '특수 기능' 버튼을 클릭하면 파일 올리기 링크가 있습니다. 그곳으로 들어간 다음 먼저 자신이 올리고 싶은 이미지 파일을 선택하고 그 이미지 파일의 '기본 정보'와 '설명'을 아래의 넓은 칸에 적습니다. 그 뒤 그 이미지 파일에 맞는 '라이선스'를 선택한 다음 파란색 '올리기' 버튼을 클릭하시면 됩니다. 이미지 파일의 이름은 편집지침을 따라주세요. 파일 이름에 _ (언더스코어)가 들어가 있다면 빼 주시고 새 문서 이름을 적용해 주세요.

파일을 올리셨다면 그 파일을 첨부하고 싶은 문서에

[[파일:example.png]]

위와 같은 형식으로 [[ ]] 사이에 파일 이름 그대로 적습니다. 그런 다음 문서를 저장하면 자동으로 올린 파일이 출력됩니다.

참고: 파일을 문서 내에 이미지 첨부 말고 그냥 파일 링크만 적고 싶을 때는 파일 이름 왼쪽에 ':'를 적어 [[:파일:example.png]] 형식으로 링크해 주십시오.

  • 앞에 1칸을 띄운 [[ 파일:example.png]] 형식도 사용 가능합니다. 이건 버그이며 조만간 수정될 예정입니다.

참고: 이 기능(링크와 출력이 다른 하이퍼링크)을 이용해 첨부한 이미지에 링크를 걸고 싶을 때에는 아래와 같이 입력해야 합니다.

[[문서|[[파일:example.png|width=너비&height=높이]]]]

이렇게 하면 다음과 같이 나옵니다: [11]

주의: 파일의 새 이름을 지정하고자 할 때에는 반드시 '파일:'로 시작해야 합니다. 즉 파일의 새 이름 칸에 '파일:example.png'로 입력해야 하며, 'example.png'로만 되어있으면 빨간색 오류 메시지가 뜹니다.
주의: 파일의 새 이름의 확장자가 원래 이미지와 일치해야 합니다. '파일:example' 또는 '파일:example.jpg'과 같이 지정되면 빨간색 오류 메시지가 뜹니다.
주의: 컴퓨터에 저장된 파일 확장자가 대문자로 되어있다면 소문자로 바꿔 주세요. 가령 원본 파일이 'example.PNG'라고 되어있다면 'example.png'로 고친 후 자체 업로더를 이용하셔야 합니다. 바꾸지 않으면 빨간색 오류 메시지가 뜹니다. 또한 확장자명이 .jpeg라면 .jpg로 바꾼 뒤 이용하셔야 합니다.

되도록이면 용량이 작은 이미지를 올리시기를 권장합니다. 서버의 부담도 줄이고 모바일 사용자의 경우 데이터도 절약하고 문서 로딩도 빨라집니다. 올리려는 이미지가 크다면 그림 압축을 추천드립니다. Microsoft Office Picture Manager의 경우, 상단의 그림 편집 → 그림 압축에 들어가셔서 이미지를 압축할 수 있습니다.

7.2. 이미지 크기 및 위치 조절[편집]

문서에 삽입된 이미지는 화면을 넘어가지 않는 한 원래 크기로 출력됩니다. 이미지 크기나 정렬을 조절하고자 하는 경우에 이미지 링크 뒤에 매개변수를 붙일 수 있습니다. 이미지의 크기를 높이 변경으로 지정하는 것은 권장되지 않습니다. 되도록이면 가로(width) 변경으로 지정해 주세요.

width=300: 가로 300px

width=25%: 화면 너비의 25%

height=300: 세로 300px

align=left: 왼쪽 정렬

align=center: 가운데 정렬

align=right: 오른쪽 정렬


또한, 표 안에 상대적으로 크기가 큰(width값이 300px 이상인) 이미지를 삽입할 때, [[파일:이미지.jpg|width=(숫자)]]를 사용하는 것은 나무위키의 엔진 버그로 인해 추천하지 않습니다. 예시 표에 이미지를 추가하려는 경우에 다음과 같이 삽입해 주세요.

||<tablewidth=900><tablealign=center><bgcolor=white> [[파일:이미지.jpg|width=100%]] ||


매개변수는 다음과 같은 형식으로 추가됩니다.

한 개의 매개변수
[[파일:example.png|width=300]]

여러 개의 매개변수
[[파일:example.png|width=300&height=300]]
[[파일:example.png|width=300&align=right]]
[[파일:example.png|width=300&height=300&align=right]]

참고: 나무위키:틀

삽입될 문서 내에 @name@ 형태의 텍스트를 입력한 뒤, 매크로를 호출할 때 [include(문서명,name=value)] 형태로 호출할 경우, 해당 위치에 value가 삽입됩니다. 이 때 하나의 value에 쉼표(,)가 포함되어 있을 경우 제대로 표시되지 않습니다. , 대신에 //,를 사용하면 해결됩니다.

주의: 인클루드된 문서를 다시 인클루드할 경우 앞서 인클루드한 내용은 보이지 않습니다. 이는 A에서 B를, B에서 A를 서로 인클루드하여 무한반복되는 사태를 방지하기 위함으로, 이를 이용하여 틀의 설명을 달거나[21] 틀에 틀을 달 수 있습니다. 예를 들어 토론이 진행 중인 틀 문서에 틀:토론 중을 달아도 그 틀을 인클루드한 문서에선 토론 중 틀이 보이지 않습니다.

출생일: 2000년 4월 5일 (18세)

15.8. 남은 날 (dday)[편집]

age 매크로와 날짜 입력 방식은 동일합니다. 이름에서 유추할 수 있듯이, 남은 날짜를 계산해줍니다. 출력 날짜 방식은 지역 표준시가 아닌 UTC+0(협정 세계시)의 규격을 따릅니다.

2019년 11월 14일까지: D-395

"오늘부터 1일" 같은 며칠째 기념일 계산도 가능합니다. 당일(dday)이 0일로 계산되기 때문에, 당일(dday)의 전날로 입력하면 됩니다.

오늘은 2014년 4월 16일 이후로 +1643일째

여기에 내용을 넣어 주세요.

  • style="border:1px solid gray;border-top:5px solid (색 코드);padding:12px"
    테두리를 지정합니다. (색 코드) 부분에 원하는 색깔의 코드를 넣어 주면 됩니다. 아래는 많이 쓰이는 코드들입니다. 또한 blue, red 같이 CSS 웹 색상 이름을 사용해도 됩니다.

    • orange
      가장 일반적인 색으로, 주의 문구에 주로 쓰입니다. 예: 틀:회원수정

    • crimson
      orange보다 조금 더 강한 경고에 주로 쓰입니다. 예: 틀:회원수정3

    • black
      crimson보다 조금 더 강한 경고에 주로 쓰입니다. 예: 틀:관리자 수정

    • navy
      black보다도 더 강한 경고에 주로 쓰입니다. 예: 틀:불법

    • gray
      폐지, 부재 등 무언가가 없다는 의미에 주로 쓰입니다. 예: 틀:폐선

    • yellowgreen, green
      편집상의 주의 문구에 주로 쓰입니다. 예: 틀:이동제한

    • #00A495
      나무위키의 공식 문서 등에 주로 쓰입니다. 예: 틀:나무위키 기본방침

  • 텍스트 크기 문법을 이용하여 제목 크기 조절이 가능합니다. {{{+1 여기에 제목을 넣어 주세요.}}} 처럼 사용할 수 있습니다.

  • 제목과 내용 사이에 한 줄을 띄워 주세요. [br][br] 처럼 줄 바꿈 매크로를 두 번 사용하면 됩니다.

  • 틀 높이: height:(수치)px

  • 틀 길이: width:(수치)px 또는 (수치)%: %의 기준은 화면의 너비.

  • 틀 길이- 최대: max-width:(수치)px 또는 (수치)%: %의 기준은 화면의 너비.

  • 글자 위치: text-align: left 또는 center 또는 right

21. 분류[편집]

문서를 분류에 넣으려면 문서의 최상단 또는 최하단에 [[분류:분류]]를 입력하면 됩니다. 분류:분류에서 적당한 분류를 찾아 문서의 최상단 또는 최하단에 입력하면 됩니다. 그러나 중간에 입력하더라도 정상적으로 뜹니다. 입력하면 문서의 최상단 또는 최하단[39]에 아래와 같이 표시됩니다.

분류: 분류


참고로, 사용자 페이지의 '스킨 설정'에서 분류 바의 표시 위치를 최상단이나 최하단으로 바꿀 수 있습니다.

[[분류:(분류명)]]으로 새 문서를 만들면 새로운 분류가 생성됩니다. 상위 분류에 추가하고 싶다면 문서와 동일하게 [[분류:(상위 분류명)]]을 입력하면 됩니다.

단, 분류를 하는 것이 아닌 분류 문서로 링크를 하는 경우에는 앞에 : 또는 빈칸을 삽입하여야 합니다. [[:분류:분류]] 또는 [[ 분류:분류]]와 같이 입력하면 분류:분류처럼 처리됩니다.


내용 누설이나 스포일러성을 띠는 분류는 편집 시 분류 문법 뒤에 #blur를 붙이면 바로 노출되는 것을 막을 수 있습니다. 예를 들어, [[분류:ABCD]]의 경우 [[분류:ABCD#blur]]와 같이 입력하고 문서를 저장하면, 문서 내의 '분류 바'에 해당 분류에만 블러(흐리게) 효과가 적용됩니다.

''분류: 분류 |


적용 시에 위와 같이 출력됩니다. 블러 효과가 적용된 분류에 마우스 커서를 가져다 대면 블러 효과 없이 나타납니다.
